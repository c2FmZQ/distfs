User set up changes

After a user is registered
  * an admin can use "distfs admin-user-quota ..." to set the user's quota.
  * an admin can use "distfs mkdir --owner=email|UserID /path" to create a directory owned by the new user.

The admin-chown and admin-chmod commands should be removed.

Admins can mkdir a directory owned by someone else
  * The admin must have write permissions on the parent inode
  * The new inode must be a directory with no children when it is created.

Client-side inode verification. These must be true:
  * SignerID is present
  * UserSig is valid and signed by SignerID
  * If GroupSig is present, it is valid and signed by GroupID's key
  * If GroupSig is absent AND SignerID != OwnerID, Type must be directory and Children must be empty.   This happens only when an admin created the inode.

Remove AuthorizedSigners from InodeClientBlob. It is useless in terms of security.
